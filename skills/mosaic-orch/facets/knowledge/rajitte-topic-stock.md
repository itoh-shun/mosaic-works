## 重複回避ルール

| 条件 | 判定 |
|---|---|
| 過去回と同じ slug | NG（再採用不可） |
| 過去回と同じ主要キーワード3つ以上一致 | 警告 → 「続報」として位置付け可能か検討 |
| 過去回の話題のその後・続報・自分で調べた更新 | OK（むしろ番組に連続性が生まれて◎） |
| 過去回で軽く触れただけのトピックを今回深堀 | OK |
| 完全に新規のニュース | OK |

## 更新運用

- save-radio-script instruction が新台本保存時に自動追記する
- フォーマット: `#### #N (YYYY-MM-DD) — タイトル` ブロックを「使用済みトピック」配下に追加
- slug は kebab-case で命名、領域別カテゴリの末尾に追記
- 候補ストックから採用したら「使用済み」に移動し、候補から削除
