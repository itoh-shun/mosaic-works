# 指示: セクション執筆

与えられたセクション情報に基づいて、技術記事の1セクションを執筆してください。

## 手順
1. セクションの aim（狙い）を確認する
2. 読者が何を知りたいかを想定する
3. 「なぜ」→「何を」→「どうやって」の順で構成する
4. 抽象的な説明にはコード例・具体例を添える
5. 想定文字数（estimated_chars）に収まるように調整する
6. 次のセクションへの自然なつなぎを意識する

## 執筆ガイドライン
- 1段落は3-5文を目安にする
- コードブロックには言語指定をつける
- 専門用語は初出時に簡潔に説明する
- 図解が有効な箇所は `<!-- 図: {説明} -->` で提案する
- 読者への問いかけや「試してみよう」を適宜入れる
